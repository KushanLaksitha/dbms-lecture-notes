Practical 01 - MySQL Basics
Q1: What is MySQL?
Explain MySQL and its characteristics.

Answer: MySQL is an open-source Relational Database Management System (RDBMS) that enables users to store, manage, and retrieve structured data efficiently.

Advantages:

  • Free and Open Source
  • Large Community for Support
  • High Performance, Scalability, Flexibility
  • Platform Independency

Disadvantages:

  • Stability issues
  • Poor Performance in High Loads
Q2: How to start working with MySQL?
List the steps to log in to MySQL.

Steps:

1. Open Command Prompt 2. Navigate to MySQL Bin Directory: cd <path to MySQL Bin Directory> 3. Log in to MySQL: mysql -u root -p 4. Enter your password 5. You'll see mysql> prompt when successfully logged in
Q3: Create a Database
How do you create a database named COMPANY?
CREATE DATABASE COMPANY;
Q4: Use a Database
How do you select a database for use?
USE COMPANY;

This tells MySQL to use the named database as the default database for subsequent statements.

Practical 02 - Constraints & SELECT
Q1: What are SQL Constraints?
List and explain SQL constraints.

SQL Constraints:

  • NOT NULL - Ensures a column cannot have a NULL value
  • UNIQUE - Ensures all values in a column are different
  • PRIMARY KEY - Combination of NOT NULL and UNIQUE. Uniquely identifies each row
  • FOREIGN KEY - Prevents actions that would destroy links between tables
  • CHECK - Ensures values in a column satisfy a specific condition
  • DEFAULT - Sets a default value for a column if no value is specified
Q2: Create Table with Constraints
Create EMPLOYEE table with NOT NULL, PRIMARY KEY, CHECK, and DEFAULT constraints.
CREATE TABLE EMPLOYEE( Fname VARCHAR(15) NOT NULL, Minit CHAR, Lname VARCHAR(15) NOT NULL, Ssn CHAR(9) NOT NULL, Bdate DATE, Address VARCHAR(30) DEFAULT 'Colombo', Sex CHAR, Age INT, Salary DECIMAL(10,2), Super_ssn CHAR(9), Dno INT NOT NULL, PRIMARY KEY (Ssn), CHECK (Age >= 18) );
Q3: Create DEPARTMENT Table
Create DEPARTMENT table with FOREIGN KEY and UNIQUE constraints.
CREATE TABLE DEPARTMENT( Dname VARCHAR(15) NOT NULL, Dnumber INT NOT NULL, Mgr_ssn CHAR(9) NOT NULL, Mgr_start_date DATE, PRIMARY KEY (Dnumber), UNIQUE (Dname), FOREIGN KEY (Mgr_ssn) REFERENCES EMPLOYEE(Ssn) );
Q4: SELECT All Records
Retrieve all data from EMPLOYEE table.
SELECT * FROM EMPLOYEE;

The asterisk (*) selects all columns

Q5: SELECT Specific Columns
Retrieve only first name and address from EMPLOYEE.
SELECT Fname, Address FROM EMPLOYEE;
Q6: SELECT DISTINCT
Retrieve unique addresses from EMPLOYEE (remove duplicates).
SELECT DISTINCT Address FROM EMPLOYEE;
Q7: WHERE Clause
Retrieve employees who live in Kandy.
SELECT * FROM EMPLOYEE WHERE Address = 'Kandy';
Q8: Multiple Conditions with AND
Retrieve birth date and address of 'John B. Smith'.
SELECT Bdate, Address FROM EMPLOYEE WHERE Fname = 'John' AND Minit = 'B' AND Lname = 'Smith';
Q9: Join Two Tables
Retrieve name and address of employees who work in 'Research' department.
SELECT Fname, Lname, Address FROM EMPLOYEE, DEPARTMENT WHERE Dname = 'Research' AND Dnumber = Dno;
Q10: Qualified Column Names
Rewrite Q9 with fully qualified attribute names to prevent ambiguity.
SELECT EMPLOYEE.Fname, EMPLOYEE.Lname, EMPLOYEE.Address FROM EMPLOYEE, DEPARTMENT WHERE DEPARTMENT.Dname = 'Research' AND DEPARTMENT.Dnumber = EMPLOYEE.Dno;
Q11: Table Aliases
Use aliases to shorten table names in queries.
SELECT E.Fname, E.Lname, E.Address FROM EMPLOYEE AS E, DEPARTMENT AS D WHERE D.Dname = 'Research' AND D.Dnumber = E.Dno;

AS keyword creates an alias (E for EMPLOYEE, D for DEPARTMENT)

Q9: Difference Between Q7 and Q8
Explain the difference in results.

Answer:

  • Q7 (without DISTINCT): Shows all city values including duplicates. Result has 5 rows with "Colombo" appearing 3 times.
  • Q8 (with DISTINCT): Shows only unique city values. Result has 2 rows with each city appearing once.
Q10: Count Distinct Cities
Count the number of unique cities.

Result: 2 (Colombo and Nugegoda)

Q11: WHERE - Single Condition
Select all customers from Colombo.
SELECT * FROM CUSTOMER WHERE City = 'Colombo';
Q12: WHERE with OR
Select customers whose occupation is "Banker" OR "Medical Officer".
SELECT * FROM CUSTOMER WHERE Occupation = 'Banker' OR Occupation = 'Medical Officer';
Q13: WHERE with NOT
Select all customers whose occupation is NOT "Banker".
SELECT * FROM CUSTOMER WHERE Occupation != 'Banker'; -- OR SELECT * FROM CUSTOMER WHERE Occupation <> 'Banker';
Q14: Complex WHERE with Parentheses
Select customers from Colombo AND occupation is "Banker" OR "Software Developer".
SELECT * FROM CUSTOMER WHERE City = 'Colombo' AND (Occupation = 'Banker' OR Occupation = 'Software Developer');
💡 Parentheses are important! They ensure OR is evaluated first, then AND.
Q15: ORDER BY Ascending
Sort customers by Occupation in ascending order.

Default order is ASC (ascending): Accountant, Banker, Banker, Medical Officer, Software Developer

Q16: ORDER BY Descending
Sort customers by Occupation in descending order.
SELECT * FROM CUSTOMER ORDER BY Occupation DESC;

DESC orders from Z to A: Software Developer, Medical Officer, Banker, Banker, Accountant

Part 2: Supplier-Parts Database
Q6: Create Supplier Table
Create Supplier table with constraints.
CREATE TABLE Supplier( Sno VARCHAR(5) PRIMARY KEY, Sname VARCHAR(30) NOT NULL, Status INT, City VARCHAR(30) );
Q7: Create Parts Table
Create Parts table with constraints.
CREATE TABLE Parts( Pno VARCHAR(5) PRIMARY KEY, Pname VARCHAR(30) NOT NULL, Color VARCHAR(20), Weight INT, City VARCHAR(30) );
Q8: Create Supply Table
Create Supply table with foreign keys.
CREATE TABLE Supply( Sno VARCHAR(5), Pno VARCHAR(5), Quantity INT, PRIMARY KEY (Sno, Pno), FOREIGN KEY (Sno) REFERENCES Supplier(Sno), FOREIGN KEY (Pno) REFERENCES Parts(Pno) );
