SQL Examples, Practice Exercises, and Software Links

SQL Examples

  • CREATE TABLE: Defines a new table.
    CREATE TABLE Students ( StudentID INT PRIMARY KEY, FirstName VARCHAR(50), LastName VARCHAR(50), DateOfBirth DATE );
  • INSERT INTO: Adds a new row of data into a table.
    INSERT INTO Students (StudentID, FirstName, LastName, DateOfBirth) VALUES (101, 'John', 'Smith', '2002-08-15');
  • SELECT: Retrieves data from one or more tables.
    SELECT FirstName, LastName FROM Students WHERE StudentID = 101;
  • UPDATE: Modifies existing records in a table.
    UPDATE Students SET LastName = 'Jones' WHERE StudentID = 101;
  • DELETE: Removes records from a table.
    DELETE FROM Students WHERE StudentID = 101;
  • JOIN: Combines rows from two or more tables based on a related column.
    SELECT Students.FirstName, Courses.CourseName FROM Students JOIN Enrollments ON Students.StudentID = Enrollments.StudentID JOIN Courses ON Enrollments.CourseID = Courses.CourseID;

Practice Exercises

  • Scenario 1: Simple Blog Database

    Design a database for a simple blog. It should have tables for `Posts` and `Comments`. What columns would you include in each table? How would you link a comment to a specific post? Think about the relationship type and cardinality.

  • Scenario 2: Library Management System

    Create an ER diagram for a library. The database should track `Books`, `Members`, and `Loans` (which book is borrowed by which member). Consider the attributes for each entity and the relationships between them. What happens when a member can borrow multiple books?

  • Scenario 3: Query Challenge

    Given a table `Products` with columns `ProductID`, `ProductName`, `Category`, and `Price`, write a SQL query to find the average price of products in the 'Electronics' category.
