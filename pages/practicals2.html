üè¶ Practical 05 - Bank Database System

Database Schema:

bank (BankCode, name, registration)

branch (BranchNumber, BankCode, name, address)

account_type (AccTypeCode, name, interest_rate, description)

account (AccountNo, balance, BankCode, BranchNumber, AccType)

customer (CusID, NIC, name, address, phone)

belongs (CustomerID, Anumber)

Step 1: Create Database and Tables

CREATE DATABASE Bank; USE Bank; CREATE TABLE bank ( BankCode VARCHAR(10) PRIMARY KEY, name VARCHAR(100), registration VARCHAR(20) ); CREATE TABLE branch ( BranchNumber INT PRIMARY KEY, BankCode VARCHAR(10), name VARCHAR(50), address VARCHAR(200), FOREIGN KEY (BankCode) REFERENCES bank(BankCode) ); CREATE TABLE account_type ( AccTypeCode VARCHAR(20) PRIMARY KEY, name VARCHAR(50), interest_rate DECIMAL(5,2), description VARCHAR(200) ); CREATE TABLE account ( AccountNo VARCHAR(20) PRIMARY KEY, balance DECIMAL(15,2), BankCode VARCHAR(10), BranchNumber INT, AccType VARCHAR(20), FOREIGN KEY (BankCode) REFERENCES bank(BankCode), FOREIGN KEY (BranchNumber) REFERENCES branch(BranchNumber), FOREIGN KEY (AccType) REFERENCES account_type(AccTypeCode) ); CREATE TABLE customer ( CusID VARCHAR(10) PRIMARY KEY, NIC VARCHAR(15), name VARCHAR(100), address VARCHAR(200), phone VARCHAR(15) ); CREATE TABLE belongs ( CustomerID VARCHAR(10), Anumber VARCHAR(20), PRIMARY KEY (CustomerID, Anumber), FOREIGN KEY (CustomerID) REFERENCES customer(CusID), FOREIGN KEY (Anumber) REFERENCES account(AccountNo) );

üîó Practical 09 - SQL Joins

Understanding Joins

What is a Join?
A join is used to query data from more than one table based on a relationship between columns.

Join Syntax Reference

-- INNER JOIN: Returns matching records from both tables SELECT columns FROM table1 INNER JOIN table2 ON table1.column = table2.column; -- LEFT JOIN: Returns all records from left table, matching from right SELECT columns FROM table1 LEFT JOIN table2 ON table1.column = table2.column; -- RIGHT JOIN: Returns all records from right table, matching from left SELECT columns FROM table1 RIGHT JOIN table2 ON table1.column = table2.column;

Step 1: Create Tables

DROP DATABASE IF EXISTS XYZ_Company; CREATE DATABASE XYZ_Company; USE XYZ_Company; CREATE TABLE Department( DeptCode INT NOT NULL, DeptName VARCHAR(15) NOT NULL, DeptHead VARCHAR(15), RegDate DATE, PRIMARY KEY (DeptCode) ); CREATE TABLE employees( EMPLOYEE_ID CHAR(3), FIRST_NAME VARCHAR(10), LAST_NAME VARCHAR(10), HIRE_DATE DATE, JOB_ID VARCHAR(10), SALARY DECIMAL(10,2), Dept_ID INT, PRIMARY KEY (EMPLOYEE_ID), FOREIGN KEY (Dept_ID) REFERENCES Department(DeptCode) );

üìù Query Solutions

2) INNER JOIN - Show only matching records
SELECT e.EMPLOYEE_ID, e.FIRST_NAME, e.LAST_NAME, e.JOB_ID, e.SALARY, d.DeptName FROM employees e INNER JOIN Department d ON e.Dept_ID = d.DeptCode;
Result: Returns only employees who have a matching department (excludes employees with NULL Dept_ID)
3) LEFT JOIN - Show all employees
SELECT e.EMPLOYEE_ID, e.FIRST_NAME, e.LAST_NAME, e.JOB_ID, e.SALARY, d.DeptName FROM employees e LEFT JOIN Department d ON e.Dept_ID = d.DeptCode;
Result: Returns ALL employees, even those without a department (shows NULL for department info)
4) RIGHT JOIN - Show all departments
SELECT e.EMPLOYEE_ID, e.FIRST_NAME, e.LAST_NAME, e.JOB_ID, e.SALARY, d.DeptName FROM employees e RIGHT JOIN Department d ON e.Dept_ID = d.DeptCode;
Result: Returns ALL departments, even those without employees (shows NULL for employee info)
5) Find employees with salary between 30000 and 50000
SELECT e.EMPLOYEE_ID, e.FIRST_NAME, d.DeptName, e.JOB_ID FROM employees e INNER JOIN Department d ON e.Dept_ID = d.DeptCode WHERE e.SALARY BETWEEN 30000 AND 50000;
üí° Join Comparison:
‚Ä¢ INNER JOIN - Only matching records
‚Ä¢ LEFT JOIN - All from left table + matches from right
‚Ä¢ RIGHT JOIN - All from right table + matches from left
‚Ä¢ FULL OUTER JOIN - All records from both tables (not in MySQL, use UNION)
