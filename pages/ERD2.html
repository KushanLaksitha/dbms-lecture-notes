ðŸ“– Scenario: University Course Management System

Greenfield University needs a comprehensive system to manage its academic operations. The university offers various courses, each identified by a unique course code and having a title, credit hours, and description.

Each course can have multiple instructors teaching different sections, and instructors can teach multiple courses. Instructors have an employee ID, name (which includes first name, middle initial, and last name), date of birth, and can have multiple phone numbers (office, mobile, home). Instructors also have an office location that consists of building name, room number, and floor.

The university maintains information about course prerequisites. Some courses require students to complete one or more other courses before enrollment. For example, "Data Structures" might be a prerequisite for "Database Systems", and "Database Systems" might be a prerequisite for "Advanced Database Management".

Students enroll in courses, and each enrollment is tracked with a specific section number and semester. The system needs to record each student's ID, name (first, middle, last), date of birth, email addresses (they can have multiple emails - personal and university), and their current GPA. When a student enrolls in a course, the system records the semester, year, and the grade received (once the course is completed).

Each course section is assigned to a specific classroom, identified by a building and room number combination. However, the classroom assignment depends on the section's existence - a classroom reference is meaningless without the associated course section. The classroom has a capacity and equipment details.

Problem: Transitive dependencies where non-key attributes depend on other non-key attributes

Solution: Remove transitive dependencies by creating separate tables

ðŸ’» SQL Implementation

Create Tables

-- Create STUDENT table
CREATE TABLE STUDENT ( StudentID VARCHAR(10) PRIMARY KEY, FirstName VARCHAR(50) NOT NULL, MiddleInitial CHAR(1), LastName VARCHAR(50) NOT NULL, DateOfBirth DATE, GPA DECIMAL(3,2) CHECK (GPA >= 0.0 AND GPA <= 4.0) );
-- Create STUDENT_EMAIL table (handles multivalued attribute)
CREATE TABLE STUDENT_EMAIL ( StudentID VARCHAR(10), Email VARCHAR(100), PRIMARY KEY (StudentID, Email), FOREIGN KEY (StudentID) REFERENCES STUDENT(StudentID) ON DELETE CASCADE );
-- Create COURSE table
CREATE TABLE COURSE ( CourseCode VARCHAR(10) PRIMARY KEY, Title VARCHAR(100) NOT NULL, CreditHours INT CHECK (CreditHours > 0), Description TEXT );
-- Create INSTRUCTOR table
CREATE TABLE INSTRUCTOR ( EmployeeID VARCHAR(10) PRIMARY KEY, FirstName VARCHAR(50) NOT NULL, MiddleInitial CHAR(1), LastName VARCHAR(50) NOT NULL, DateOfBirth DATE );
-- Create INSTRUCTOR_PHONE table (handles multivalued attribute)
CREATE TABLE INSTRUCTOR_PHONE ( EmployeeID VARCHAR(10), PhoneNumber VARCHAR(15), PhoneType VARCHAR(20), -- 'Office', 'Mobile', 'Home' PRIMARY KEY (EmployeeID, PhoneNumber), FOREIGN KEY (EmployeeID) REFERENCES INSTRUCTOR(EmployeeID) ON DELETE CASCADE );
-- Create INSTRUCTOR_OFFICE table (handles composite attribute)
CREATE TABLE INSTRUCTOR_OFFICE ( EmployeeID VARCHAR(10) PRIMARY KEY, BuildingName VARCHAR(50), RoomNumber VARCHAR(10), Floor INT, FOREIGN KEY (EmployeeID) REFERENCES INSTRUCTOR(EmployeeID) ON DELETE CASCADE );
-- Create ENROLLMENT table (many-to-many relationship)
CREATE TABLE ENROLLMENT ( StudentID VARCHAR(10), CourseCode VARCHAR(10), Semester VARCHAR(10), -- 'Fall', 'Spring', 'Summer' Year INT, SectionNumber INT, Grade CHAR(2), -- 'A', 'B+', 'C', etc. PRIMARY KEY (StudentID, CourseCode, Semester, Year, SectionNumber), FOREIGN KEY (StudentID) REFERENCES STUDENT(StudentID) ON DELETE CASCADE, FOREIGN KEY (CourseCode) REFERENCES COURSE(CourseCode) ON DELETE CASCADE );
-- Create TEACHES table (many-to-many relationship)
CREATE TABLE TEACHES ( EmployeeID VARCHAR(10), CourseCode VARCHAR(10), SectionNumber INT, Semester VARCHAR(10), Year INT, PRIMARY KEY (EmployeeID, CourseCode, SectionNumber, Semester, Year), FOREIGN KEY (EmployeeID) REFERENCES INSTRUCTOR(EmployeeID) ON DELETE CASCADE, FOREIGN KEY (CourseCode) REFERENCES COURSE(CourseCode) ON DELETE CASCADE );
-- Create PREREQUISITE table (recursive relationship)
CREATE TABLE PREREQUISITE ( CourseCode VARCHAR(10), PrerequisiteCourseCode VARCHAR(10), PRIMARY KEY (CourseCode, PrerequisiteCourseCode), FOREIGN KEY (CourseCode) REFERENCES COURSE(CourseCode) ON DELETE CASCADE, FOREIGN KEY (PrerequisiteCourseCode) REFERENCES COURSE(CourseCode) ON DELETE CASCADE, CHECK (CourseCode != PrerequisiteCourseCode) );
-- Create CLASSROOM table (weak entity - depends on course section)
CREATE TABLE CLASSROOM ( CourseCode VARCHAR(10), SectionNumber INT, Semester VARCHAR(10), Year INT, Building VARCHAR(50), RoomNumber VARCHAR(10), Capacity INT, Equipment TEXT, PRIMARY KEY (CourseCode, SectionNumber, Semester, Year), FOREIGN KEY (CourseCode, SectionNumber, Semester, Year) REFERENCES TEACHES(CourseCode, SectionNumber, Semester, Year) ON DELETE CASCADE );
